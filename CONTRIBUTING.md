# Contributing to lmsr-lab

Thank you for your interest in contributing to **lmsr-lab**!

This project is maintained by **Robert Thijs Kozma** in a personal capacity. It is developed primarily during free time and released as open source so that others (including companies) can use and build upon it.

## How to Contribute

We welcome contributions in many forms:

- Bug reports and feature requests
- Code contributions (bug fixes, new features, improvements)
- Documentation improvements
- Example trade histories or analysis scripts
- Feedback on the architecture and design

## Contribution Process

1. **Fork** the repository and create your branch from `main`.
2. **Make your changes** and ensure they are well-tested and documented.
3. **Open a Pull Request** against the `main` branch.
4. **Sign the Contributor License Agreement (CLA)** if requested (see below).

We use the standard GitHub flow. Please keep pull requests focused and reasonably sized.

## Code of Conduct

Be respectful and constructive. We follow the standard [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

## Questions?

If you have questions about contributing, feel free to open an issue or reach out to the maintainer.
